## Feature coverage

- Manual picker-header save reads PNG or bitmap clipboard images and stores them in Captures.
- Automatic saving remains off by default and can be enabled in picker settings.
- File-copy clipboard formats are excluded from capture storage.
- SHA-256 content hashes prevent duplicate capture entries.
- Metadata and finalized assets roll back cleanly when persistence fails.
- Automatic capture errors are isolated from normal clipboard compression and surfaced by toast.
- Picker state, localized labels, settings text, drop placeholders, and paste behavior are covered.
- Existing text paste behavior remains available when the clipboard has no image MIME payload.

## Environment limitations

- Inno Setup 6 was not installed locally, so `NotroSetup.exe` could not be compiled in this
  workstation QA pass. The repository release workflow remains responsible for producing and
  attaching the installer from `installer.iss`.
- Interactive picker visual QA could not run in the Codex in-app browser because its security
  policy rejects local `file://` pages. DOM structure and JavaScript syntax were verified, and
  GUI launch was intentionally avoided to prevent interference with the user's installed
  single-instance application and live clipboard.
